# W7D3 RAG 2.0 — Before / After RAGAS Evaluation

This report documents the retrieval-quality effect of the W7D3 features
(hybrid dense + FTS, RRF fusion, MMR diversification, BGE reranking, and
metadata filtering) versus the W7D2 dense-only baseline.

## Evaluation setup

* Golden set: `tests/golden/expense_golden_50.jsonl` (55 rows, well above
  the 50-row minimum required by the assignment).
* Per the W7D3 cohort amendment (posted 2026-07-20), RAGAS
  `evaluate()` is run on a **15-row deterministic subset** sampled evenly
  from the 50-row golden set (see `tests/test_ragas_gate.py::
  _deterministic_subset`). The 50-row shape gate remains active.
* Faithfulness gate: **>= 0.85**. Any faithfulness cell below 0.85 is a
  ship-blocker.
* Soft floors: answer_relevancy 0.80, context_precision 0.75, context_recall
  0.80.
* Retrieval variants exercised: baseline (W7D2), + hybrid, + rerank, + MMR,
  + metadata filter, and all-on.

## Attribution — what each stage should shift

* **hybrid (dense + Postgres FTS + RRF fusion)** — improves *context_recall*
  most, because Postgres FTS surfaces exact phrase or product-code hits
  (e.g., "Schedule C line 22") that dense retrieval alone can miss.
  Rank-based RRF avoids the fragile score-averaging trap.
* **BGE cross-encoder rerank** — improves *context_precision* and, through
  it, *faithfulness*: the reranker demotes plausible-looking but
  off-topic dense hits.
* **MMR diversification (lambda = 0.7)** — improves *answer_relevancy* on
  multi-aspect questions by preventing the top-k from collapsing into
  near-duplicate passages of the same document.
* **Metadata filtering (`chunk_metadata @> {...}::jsonb`)** — the largest
  compliance win: it enforces category/tenant scoping in the retrieval SQL
  itself (backed by the per-tenant partial HNSW indexes from
  `V002__rag2_metadata_and_partial_indexes.sql`), which is what makes
  DB-side tenant isolation checkable by
  `tests/test_tenant_isolation.py`.

## Faithfulness < 0.85 policy

Any faithfulness cell below 0.85 is flagged and blocks release.
`tests/test_ragas_gate.py::test_ragas_faithfulness_meets_gate` raises
`SystemExit` on breach so CI halts rather than silently degrading. Once the
external evaluator runs successfully, we will annotate each cell that dips
below 0.85 with the failure reason and paste the RAGAS report link.
